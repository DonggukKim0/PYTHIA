파티아 실행 결과에서 마지막 pthat 구간의 ROOT 파일에 엔트리가 0인 이유는 분석 코드의 pTHat 조건문 때문입니다. 현재 pythia.C:38-64에서 모든 이벤트에 대해 `if (pTHat >= pTHatMin && pTHat < pTHatMax)` 검사가 이루어지는데, 가장 마지막 구간은 설정 파일(config_files_13_6_TeV/pythia_config_13_6_TEV_pthat_234_infy.cmnd:36)에서 `PhaseSpace:pTHatMax = -1`로 지정되어 있어 사실상 상한이 해제된 상태입니다. 하지만 코드의 조건은 pTHat이 pTHatMax보다 작은 상황만 허용하기 때문에 pTHatMax가 음수(-1)인 경우 어떤 이벤트도 통과하지 못합니다. 그 결과 hnevent와 다른 히스토그램이 채워지지 않아 ROOT 파일이 모두 0으로 남습니다. 다른 pthat 구간은 양수 상한을 가지고 있어 문제가 발생하지 않습니다. 따라서 마지막 구간도 처리하려면 조건을 `if (pTHat >= pTHatMin && (pTHatMax < 0 || pTHat < pTHatMax))`처럼 바꿔 pTHatMax가 음수인 경우 상한 검사를 건너뛰도록 수정하면 됩니다.
